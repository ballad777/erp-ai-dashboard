# 智能金融資料分析：產品體驗稽核

稽核日期：2026-06-14  
稽核範圍：總覽、資料工作區、模型實驗室、金融分析、報告中心，以及 390px 手機版主要入口。  
主要目標：讓第一次使用者理解產品價值、完成資料匯入，並清楚感受到分析正在真實執行。

## 稽核步驟

### 1. 桌面版總覽空狀態

截圖：`截圖/01-桌面版-總覽空狀態.png`  
健康度：需要重整

優點：

- 左側五項導覽清楚，使用者能理解產品主要模組。
- 主要動作「加入第一份資料」明確。
- 畫面沒有展示假數據，符合產品可信度原則。

問題：

- 這不是傳統 Hero，而是工作台頁首；標題只有操作主張，沒有回答產品能替誰解決什麼問題。
- 頁首和空狀態同時提供前往資料頁的 CTA，形成重複選擇。
- 大面積空白沒有轉化成引導、範例或價值說明，第一印象接近尚未完成的後台。
- 品牌標誌只有「智」字，辨識度偏低，無法形成成熟 SaaS 的記憶點。

截圖：`截圖/02-桌面版-資料工作區.png`

## Typography

目前桌面頁首 H1 約 36px，手機約 27px；內文約 14–15px，eyebrow 與部分輔助文字約 10px。

建議：

- 若首頁保留為產品工作台：H1 維持 36–40px，重點應放在資料狀態與下一步，而不是無限制放大。
- 若新增對外品牌首頁：Hero title 可使用 56–68px，手機 38–44px。
- 內頁 H1 統一 32–36px；區塊標題 20–24px；一般內文 15–16px。
- 將 10px eyebrow 和次要文字提高到至少 12px，避免高解析螢幕上過度纖細。
- 中文模式不要保留 `WORKSPACE OVERVIEW`、`MODEL LAB` 等英文 eyebrow；應完整翻譯，英文模式再整體切換。
- 漸層文字只用於品牌首頁的一個關鍵短語，不應套用到資料或分析結果。

## 背景與動態

目前已有 24–29 秒 ambient field、游標光暈、grain、頁面淡入和 reduced-motion 支援，技術方向正確，但大部分效果被大面積白色工作面覆蓋，使用者幾乎感受不到層次。

建議：

- 背景保留低對比 mesh 與極細 grain，但只在首頁 Hero、空狀態和主要轉場可見。
- 不新增粒子、霓虹球或高亮掃光。
- 讓工作表面透過邊界、陰影和局部材質形成前景／中景／背景，而不是增加更多卡片。
- 頁面切換採 180–240ms；大型結果區採 320–420ms progressive reveal。
- 只在可互動表面使用游標感應光，資料表和長文字區保持靜止。

## 微互動

目前已有按鈕、route reveal、toast、details 展開與結果淡入，但互動多屬通用樣式，尚未形成產品個性。

建議：

- 上傳成功後，檔案列逐筆確認，而不是整區瞬間出現。
- 模型完成後先顯示「最佳結果」，其餘比較資料延後 80–120ms 分段揭露。
- 導覽切換可保留 active indicator 的共享位置轉場。
- 主要按鈕 hover 只做 1px 位移、邊框和陰影變化，不做誇張縮放。
- 分析設定改變時，立即更新一行「系統將執行什麼」的摘要，增加控制感。

## 音效設計

目前沒有音效。

適合加入：

- 分析完成：極短、柔和的完成提示。
- 上傳或報告成功：比完成音更輕的確認音。
- 需要使用者處理的錯誤：低音量、非警報式提示。

不適合加入：

- hover 音效。
- 每個按鈕都播放 click 音。
- 分析期間循環播放聲音。
- 自動播放或預設開啟音效。

產品規則：

- 預設關閉，在設定中提供「介面音效」開關。
- 遵守瀏覽器自動播放限制、系統靜音與使用者偏好。
- 完成音只在超過數秒的背景任務結束時播放，避免頻繁干擾。

## 分析流程

目前 loading 元件會每 1.4 秒輪播三段文字，模型、金融、報告也各有不同文案。這比單一 spinner 好，但輪播不是後端真實進度，不能包裝成已完成的步驟。

建議的真實流程：

1. 已接收資料。
2. 正在建立資料輪廓。
3. 正在判斷問題類型與候選模型。
4. 正在訓練與評估。
5. 正在產生洞察和圖表。
6. 已完成，可先看最佳結論。

需要後端提供 job id、目前 stage、已完成模型數與錯誤狀態。若尚未提供真實進度，只能使用「正在處理」和活動紀錄，不能顯示虛假的百分比。

## 語言切換

建議預設語言為繁體中文，提供完整英文切換。

範圍必須包含：

- 導覽、標題、按鈕、空狀態、toast、錯誤與 loading。
- 後端錯誤訊息。
- AI 摘要、Word 報告與圖表標題。
- 日期、數字、百分比和幣別格式。
- 頁面 metadata 與無障礙文字。

不應翻譯：

- 使用者資料集原始欄位名稱。
- 檔名、模型 key、程式碼識別字。

介面位置：

- 桌面版放在右上角帳號／設定區，以「繁中 / EN」或語言圖示開啟選單。
- 手機版放進設定或更多選單，不占用五項核心底部導覽。
- 偏好應保存在 cookie 或使用者設定，重新開啟仍維持選擇。

## 優先順序

### P0：產品結構

1. 決定首頁是對外品牌落地頁，還是登入後工作台。
2. 建立真正的繁中／英文內容架構。
3. 將分析進度連接後端真實 stage。
4. 移除重複 CTA，讓每個畫面只有一個主要下一步。

### P1：資訊與品牌層級

1. 為模型、金融、報告建立不同的空狀態與產品預覽。
2. 建立首頁專屬 typography，不把 Hero 字級套到所有工具頁。
3. 強化品牌標誌、產品敘事和可信證據。
4. 將報告中心補成可回顧成果的真正中心。

### P2：質感細節

1. 重新分配 ambient background 的可見位置。
2. 加入真實狀態驅動的 progressive reveal。
3. 統一 hover、press、focus、success、error 動態。
4. 加入預設關閉的極簡完成音效。

## 證據限制

- 本次 Browser 能完整擷取空工作區與 RWD，但沒有可用的瀏覽器檔案注入介面，因此未擷取真實上傳後的結果畫面。
- loading、reduced motion、ambient background 與音效狀態另外依本次程式碼檢查確認。
- 截圖可確認視覺風險，但不能單獨證明完整 WCAG 相容性；鍵盤順序、螢幕閱讀器和實際對比仍需專項測試。
