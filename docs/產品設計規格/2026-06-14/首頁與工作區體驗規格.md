# 智能金融資料分析：首頁與工作區體驗規格

日期：2026-06-14  
狀態：視覺方向已確認，尚未進入程式實作  
選定視覺：`首頁選定視覺方向.png`

## 1. 產品核心

### 核心主張

> 把複雜資料，變成清楚的下一步。

產品不替使用者做決策，而是：

- 幫助使用者理解資料。
- 找出值得先看的重點。
- 說明分析結果的依據與限制。
- 降低模型與金融分析門檻。
- 提供可執行的下一步方向。

### 核心敘事

> 理解資料 → 找到重點 → 看見依據 → 知道下一步

這段敘事應同時出現在：

- 品牌首頁。
- 工作區總覽。
- 分析進度。
- 模型與金融結果。
- AI 建議。
- 報告與程式碼輸出。

## 3. 共用設計語言

首頁與工作區共用：

- 色彩 tokens。
- 字體家族與數字格式。
- 6–12px 控制元件圓角。
- 邊框、陰影、focus、success、warning、error 語意。
- 180–240ms 微互動。
- 語言與音效偏好。

首頁差異：

- 更大的文字尺度。
- 更寬鬆的垂直節奏。
- 更明顯的品牌敘事。
- 允許低透明度背景材質與捲動揭露。
- 產品畫面作為主要視覺證據。

工作區差異：

- 更高資訊密度。
- 更短的頁面標題。
- 導航與操作位置穩定。
- 動畫只用於狀態、因果與漸進揭露。
- 表格、圖表與長文字區保持安靜。

## 4. 品牌首頁規格

### 4.1 Header

桌面版：

- 高度：64–68px。
- 最大內容寬度：1280–1360px。
- 左側：品牌標誌與「智能金融資料分析」。
- 中間：產品能力、分析流程、資料與安全。
- 右側：語言切換、登入或工作區入口、主要 CTA。

主要 CTA：

> 開始分析資料

語言選單：

> 繁中 / EN

捲動後：

- Header 保持 sticky。
- 背景從透明轉為 92–96% 白色。
- 僅增加 1px 底線與低陰影，不做強烈毛玻璃。

手機版：

- Logo、語言按鈕、選單按鈕。
- 主要 CTA 放入選單底部。
- 選單使用 sheet，不使用多層下拉。

### 4.2 Hero

桌面高度：

- 最小 760px。
- 最大不超過 920px。
- 所有常見桌面高度都要露出下一個 Section 的開頭。

文案：

**Eyebrow**

> 智能金融與資料分析工作區

**H1**

> 把複雜資料，  
> 變成清楚的下一步。

**Supporting copy**

> 上傳 CSV 或 Excel，系統會理解資料結構、找出關鍵重點、推薦合適的分析方式，並將結果整理成圖表、程式碼與報告。

**Primary CTA**

> 開始分析資料

**Secondary CTA**

> 了解分析流程

**Trust line**

> 真實執行 Python 分析 · 支援多檔合併 · 不使用示範結果

Typography：

- H1 桌面：64–68px，行高 1.06–1.1。
- H1 平板：48–56px。
- H1 手機：40–44px。
- H1 字重：650–720。
- Supporting copy：17–19px，行高 1.65。
- 內文最大寬度：620px。
- 只強調「清楚的下一步」，使用低飽和藍綠文字效果。

產品預覽：

- 使用真實 React 元件和真實產品狀態。
- 不使用預先合成的假 Dashboard 截圖。
- 桌面顯示：
  - 目前資料來源。
  - 建議下一步。
  - 推薦模型。
  - 最佳結果。
  - 一張金融圖表。
  - 最近輸出。
- Hero 首屏只顯示摘要，不顯示完整設定與大型表格。
- 預覽內容密度應比選定視覺稿降低約 15–20%。

背景：

- 使用低透明度座標線、圖表軸線與資料關係。
- 不得穿過主要文案閱讀區。
- 禁止粒子、霓虹球、強光掃描與紫色 AI 漸層。

### 4.3 分析旅程

標題：

> 從資料到下一步，一條清楚的分析旅程。

流程：

```text
加入資料 → 理解結構 → 選擇方法 → 找到重點 → 完成交付
```

桌面：

- 使用一條連續水平流程。
- 一次只強調一個步驟。
- 步驟切換時，下方顯示一小段真實產品狀態。
- 不使用五張等寬功能卡。

手機：

- 改為垂直 stepper。
- 當前步驟展開，其他步驟只顯示標題。
- 不做橫向滑動輪播。

### 4.4 資料理解

標題：

> 先理解資料，再選擇方法。

說明：

> 系統會辨識欄位型別、缺失值、資料品質與可能的分析目標，讓你在執行模型前先知道資料是否適合。

產品證據：

- 欄位型別。
- 缺失值摘要。
- 資料品質。
- 建議目標欄位。
- 單檔與合併資料來源。

主要連結：

> 查看資料理解流程

### 4.5 模型選擇

標題：

> 選擇合適模型，而不是執行更多模型。

說明：

> 系統會依問題類型、資料規模、特徵與缺失情況推薦候選模型；你仍可以手動調整分析模式與模型組合。

產品證據閱讀順序：

1. 問題類型判斷。
2. 推薦模型與原因。
3. 最佳模型。
4. 主要指標。
5. 完整比較。

禁止：

- 首屏直接顯示十多個模型表格。
- 使用「AI 已為你選出完美模型」等絕對文案。

### 4.6 金融洞察

標題：

> 看見結果背後的依據。

說明：

> 當資料包含日期與價格欄位，系統會整理趨勢、報酬、波動與技術指標，並清楚標示限制與風險。

產品證據：

- 價格與移動平均。
- 波動率。
- RSI。
- MACD。
- VaR。
- 一句可讀洞察。

金融分析是條件式能力，不應在首頁壓過一般資料分析定位。

### 4.7 交付成果

標題：

> 讓洞察成為下一個動作。

內容：

- 頁內 Python 程式碼預覽。
- Notebook 章節預覽。
- Word 報告預覽。
- 模型結果與圖表輸出。

文案：

> 不只看見結果，也能取得重現分析所需的程式碼、Notebook 與報告。

CTA：

> 查看可交付成果

### 4.8 真實分析與資料處理

標題：

> 真實分析，清楚交代。

內容只陳述已實作能力：

- 後端真實執行 Python。
- 模型結果依目前資料重新計算。
- 沒有示範數據代替使用者結果。
- 產出檔案的保存與限制。
- 未設定外部 LLM 時，明確標示本機規則摘要。

不得宣稱：

- 企業級安全。
- 完全匿名。
- 絕不保存資料。
- AI 保證正確。

除非後端真的具備對應能力。

### 4.9 Final CTA

標題：

> 從你的資料開始，找到下一步。

Supporting copy：

> 上傳第一份資料，先理解結構，再決定最適合的分析方式。

CTA：

> 開始分析資料

此區塊使用完整頁面 band，不放入浮動卡片。

## 5. 工作區規格

### 5.1 App Shell

桌面：

- 左側導覽寬度：220–236px。
- Topbar 高度：56–60px。
- 主內容最大寬度：1540px。
- 左側導覽：
  - 總覽。
  - 資料。
  - 模型。
  - 金融。
  - 報告。
- Topbar：
  - 頁面名稱。
  - 目前資料來源。
  - 語言。
  - 設定。

手機：

- 底部保留五項主要導覽。
- 語言與音效移入設定 sheet。
- 底部導覽需支援 safe-area。
- 主內容底部保留至少 84px。

### 5.2 `/app` 工作區總覽

無資料的新使用者：

- 不先顯示空 Dashboard。
- 顯示資料匯入任務。
- 提供格式、限制與隱私說明。
- 只有一個主要 CTA：「加入第一份資料」。

有資料的回訪使用者：

第一屏順序：

1. 建議下一步。
2. 目前資料品質。
3. 分析進度。
4. 已完成成果。

第二屏：

1. 最佳模型摘要。
2. 主要金融訊號。
3. 最近報告與程式碼。
4. 目前資料來源。

### 5.3 下一步建議元件

元件結構：

```text
建議下一步
動作標題
為什麼建議
預期得到什麼
需要注意什麼
[主要 CTA] [查看其他方向]
```

狀態規則：

| 狀態 | 建議下一步 | 主要 CTA |
|---|---|---|
| 無資料 | 加入第一份資料 | 加入資料 |
| 資料未讀取 | 建立資料輪廓 | 讀取全部 |
| 品質問題嚴重 | 先檢查缺失值與欄位型別 | 查看資料品質 |
| 尚未選目標 | 確認分析目標 | 選擇目標欄位 |
| 可模型分析 | 比較適合的模型 | 執行模型比較 |
| 有金融欄位 | 查看金融訊號 | 執行金融分析 |
| 模型已完成 | 查看最佳結果與依據 | 查看最佳結果 |
| 已有主要洞察 | 整理分析報告 | 產生報告 |
| 已有報告 | 回顧成果或分析新資料 | 查看最新成果 |

建議必須由真實狀態產生，不使用固定假推薦。

### 5.4 資料頁

第一屏：

- 上傳區。
- 已加入檔案。
- 真實處理進度。
- 合併方式。

讀取完成後：

- 先顯示資料品質與建議目標。
- 詳細欄位與數值摘要放進 tabs 或 disclosure。
- 單檔／合併來源切換位置固定。

### 5.5 模型頁

設定區：

- 目標欄位。
- 自動／手動分析模式。
- 系統將執行什麼的摘要。
- 進階設定預設收合。

結果區：

1. 最佳結果。
2. 為什麼推薦。
3. 關鍵指標。
4. 圖表。
5. 完整模型比較。
6. 程式碼與輸出。

### 5.6 金融頁

結果區：

1. 目前趨勢與主要訊號。
2. 風險摘要。
3. 價格與指標圖表。
4. 指標明細。
5. 預測與限制。
6. 下一步建議。

所有洞察需要使用保守語氣：

- 「目前資料顯示」。
- 「在本次觀察區間」。
- 「此結果不構成投資建議」。

### 5.7 報告頁

報告中心需包含：

- 最新分析摘要。
- 最近生成報告。
- 報告語言。
- 產生時間。
- 對應資料來源。
- 頁內預覽。
- 重新生成與下載。

若目前沒有持久化報告歷史，不得顯示假的歷史列表。

## 6. 分析結果閱讀順序

所有分析結果使用相同敘事：

1. **你現在可以知道什麼**
2. **最重要的發現**
3. **支持這個結論的依據**
4. **限制與注意事項**
5. **建議下一步**
6. **完整技術細節**

範例：

> **目前表現最佳：Gradient Boosting**  
> 在目前測試資料中，它的預測誤差最低，且整體表現相對穩定。

接著顯示：

- R²、RMSE、MAE。
- 訓練時間。
- 與其他模型的差異。
- 特徵重要性。
- 預測與實際值。
- 殘差。

## 9. Motion 規格

### 9.1 Tokens

```text
fast:      160–200ms
standard:  220–280ms
section:   360–440ms
ambient:   20–30s
ease-out:  cubic-bezier(0.2, 0.8, 0.2, 1)
spring:    cubic-bezier(0.16, 1, 0.3, 1)
```

### 9.2 首頁

- Header：背景與底線 180ms。
- Hero copy：一次淡入，不逐字動畫。
- 產品預覽：隨捲動上移 12–20px。
- 分析旅程：active indicator 220ms。
- Section reveal：opacity + 12px 位移，400ms。
- 不同 Section 不全部套用 stagger。

### 9.3 工作區

- 導覽 active indicator：180–220ms。
- 按鈕 hover：上移 1px。
- 按鈕 press：回到 0 或下移 1px。
- 上傳檔案列：逐筆 60–90ms。
- 最佳結果：先顯示。
- 其他結果：延後 80–120ms。
- 圖表切換：短 crossfade，不做大幅滑動。
- Disclosure：180–240ms。
- Toast：180–220ms。

### 9.4 Reduced Motion

- 關閉持續背景動畫。
- 關閉 parallax。
- 關閉 stagger 與位移。
- 保留立即 opacity 狀態變化。
- 不用動畫延遲操作或資訊出現。

## 10. 音效規格

預設：關閉。

設定名稱：

> 介面音效

允許播放：

| 事件 | 音效 |
|---|---|
| 多檔讀取完成 | 輕確認 |
| 超過 3 秒的模型分析完成 | 完成 |
| 報告生成完成 | 完成 |
| 需要使用者處理的錯誤 | 柔和錯誤 |

禁止播放：

- Hover。
- 導覽切換。
- 一般按鈕 click。
- 圖表 tooltip。
- 分析期間循環聲。
- 頁面載入。

技術規則：

- 只在使用者曾互動後播放。
- 遵守瀏覽器 autoplay 限制。
- 同一任務只播放一次。
- 分頁不在前景時不播放，或只在使用者允許時播放。
- 音量固定低於一般媒體音量。

## 11. Accessibility

- 文字與背景至少符合 WCAG AA。
- 一般文字不低於 14px；主要閱讀文字以 15–17px 為主。
- 所有 focus-visible 必須清楚。
- 動態更新使用 `aria-live`，但避免每個進度訊息都打斷螢幕閱讀器。
- 圖表提供摘要與表格替代。
- 語言切換後更新 `html lang`。
- 音效不能是唯一狀態提示。
- 所有動畫支援 `prefers-reduced-motion`。
- 手機觸控目標至少 44 × 44px。
- 200% 縮放時不得出現水平溢位。

## 12. Responsive

### Desktop：≥ 1200px

- 完整行銷導覽。
- Hero 產品預覽顯示多個摘要區。
- App 使用固定左側欄。

### Tablet：768–1199px

- Header 收斂。
- Hero 文字和產品預覽上下排列。
- App 側欄可收合。
- 圖表和表格優先保持可讀寬度。

### Mobile：≤ 767px

- Hero 文字先行。
- 產品預覽只顯示「下一步、最佳結果、圖表」三個摘要。

- App 使用底部五項導覽。
- 表格使用局部橫向捲動並提供欄位摘要。

## 13. 實作順序

### Phase A：架構

## 14. 驗收標準

- `/` 是完整品牌首頁，不出現工作區側欄。
- `/app` 是完整工作區，不出現行銷 Hero。
- 所有首頁 CTA 都導向真實功能。
- 首頁產品預覽由真實元件構成。
- 預設繁中，英文可完整切換。
- 使用者欄位名稱不被翻譯。
- 首頁與 App 共用品牌 tokens，但資訊密度清楚區隔。
- 無假分析結果、假百分比或假進度。
- 分析結果先顯示結論，再顯示證據與完整細節。
- 音效預設關閉，且不是唯一回饋。
- 390px、768px、1440px 無文字遮擋與頁面水平溢位。
- 200% 縮放仍可完成核心流程。
- `prefers-reduced-motion` 下不影響操作。
